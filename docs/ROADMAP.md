# Foundry Roadmap

This roadmap describes how Foundry evolves as a framework.

The milestones are organized around architectural capabilities rather than implementation details.

---

# Milestone 1 · Language ✅

Establish the vocabulary used throughout Foundry.

Completed:

- Event
- Observation
- Decision
- Agent
- Episode
- World

Outcome:

Foundry now has a shared language for describing objective reality, perceived information, and decision making.

---

# Milestone 1.5 · Consolidation 🚧

Capture the project's identity before expanding the engine.

Goals:

- Establish Foundry principles
- Rewrite the roadmap
- Document open architectural questions
- Describe the core architecture
- Review architectural decisions

Outcome:

The repository becomes the authoritative source for both implementation and intent.

---

# Milestone 2 · Change

Teach worlds how to evolve.

Possible concepts:

- World transitions
- Decisions
- Processes
- Time
- State evolution

Question:

What transforms one world into another?

---

# Milestone 3 · History

Represent what happened.

Possible concepts:

- Memory
- Replay
- Recorder
- Episodes

Question:

How is history preserved?

---

# Milestone 4 · Experiments

Execute many histories.

Possible concepts:

- Simulation
- Runner
- Experiment
- Batch execution

Question:

How do we evaluate many possible futures?

---

# Milestone 5 · Domains

Build real applications using Foundry.

Potential domains:

- Powerball
- Poker
- Robotics
- Markets
- Healthcare
- Traffic systems

Outcome:

Demonstrate that the same abstractions naturally support very different worlds.

---

# Guiding Principle

Foundry grows by adding concepts only after they are understood.

Every milestone should reduce architectural uncertainty before increasing implementation complexity.
